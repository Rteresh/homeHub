# Инструкции для Codex

## Область проекта

- Рабочая область проекта: `/Users/rteresh/VS code/homeHub`.
- Git-корень находится выше, в `/Users/rteresh`, поэтому не считать весь домашний каталог частью проекта.
- Основные редактируемые файлы находятся в `agents/`.
- `.venv/` является локальным виртуальным окружением; не редактировать его вручную и не использовать как источник проектной логики.

## Язык и стиль

- Отвечать пользователю на русском языке.
- Для новых или изменяемых комментариев в коде/конфигурации пояснять, что делает переменная, функция или блок, какие данные принимает и зачем нужен.
- Не добавлять широкие стилистические переписывания без явной задачи.

## Формат файлов `agents/`

- Файлы в `agents/` описывают профили агентов в TOML-подобном формате:
  - `name = "..."` задает имя профиля.
  - `description = "..."` описывает, когда профиль применять.
  - `developer_instructions = """..."""` содержит рабочие инструкции.
- При добавлении нового профиля сохранять существующую структуру и тон: четкая область ответственности, рабочий режим, фокус, проверки качества и формат возврата.
- Если профиль требует внешней документации, предпочитать официальные источники и явно отделять подтвержденные факты от предположений.

## Проверки

- Автоматизированная тестовая команда в проекте пока не обнаружена.
- Для изменений в профилях проверять как минимум:
  - файл читается как текст без случайных обрывов тройных кавычек;
  - имя и описание соответствуют назначению профиля;
  - инструкции не дублируют существующие профили без необходимости.

## Осторожность с Git

- Из-за Git-корня в домашнем каталоге `git status` может показывать множество файлов за пределами `homeHub`.
- Перед выводами о состоянии проекта ограничивать анализ файлами внутри этой рабочей области.
